Astarisx.extend

Creates and returns a shallow copy of an object and merges the properties of any subsequent objects passed in. Arguments that are null or undefined are ignored.

Properties of preceding objects will be completely overwritten by a property with the same key in subsequent objects. The values are not merged.

Usage

Astarisx.extend(obj[, objN])

Arguments

Param Type Details
obj Object An object that provides the initial properties to a shallow copy of the object.
objN (optional) Object Additional objects containing properties to merge into the target object.

Returns

Object - Shallow copy of merged object

Enables automatic handling of pushState for <a> tags. There is no need to include this mixin in order to use the built-in pushState functionality. This mixin is only required if pushState is enable and <a> tags are used in the UI for navigation.

  render: function(){...}
});

mediaChangeHandler

Triggered for registered media queries, whenever a query notification is raised. To register a media query you must specify an id within the @madia query attribute. The format of the id is .media#[id], replacing [id] with whatever identifier you would like to reference that query. See Registering a media query in the Developers Guide for more information.

Usage

mediaChangeHandler: function(id, mql, initializing){
  if(id === "tinyScreen" && mql.matches){
    this.setState({mql:mql});
  }
}

Arguments

Param Type Details
id String Identifier associated with the calling media query.
mql Object MediaQueryList Object.
initializing Boolean true if initializing otherwise undefined.

Scope

setState

Triggers a state transition.

Signature

void <= setState([nextState, otherNextState, remember, callback, delay])

Arguments

Param Type Details
nextState (optional) Object Object containing the next state values.
otherNextState (optional) Object Object containing the next state values for other data contexts. Only available in ViewModels
remember (default = true) Boolean false = Force the state controller to not retain previous state during this setState transition. Only available in the ControllerViewModel and ViewModels
callback (optional) Function Used to do sequential setState calls and is passed an error or the latest Domain Data Context state. i.e. function(err, appCtx){…}
delay (optional) Number The number of milliseconds (thousandths of a second) that the callback function call should be delayed by. A callback must be specified. This will not batch updates.

If no arguments are passed in, then the latest Domain Data Context is returned to the View.

If the arguments include a callback, setState must also be called in the callback for the View to recieve the Domain Data Context.

Usage

this.setState({ current: void(0) }, {
              busy: false,
              person: { online: true }
            }, function(appContext){
              ...
              //setState must be called to update View.
              // It can be called without arguments eg. this.setState();
              this.setState({ current: aPerson });  
            }.bind(this));

OR with delay set

this.setState({fadeOut: true}, function(){
                this.setState({currentView: View});
            }.bind(this), 300); //delay callback by 300 milliseconds

kind

Assigns a field type.

Attributes

Value Type Details
uid String Identifies the unique identifier for the Model. Only available within a Model and should only specified once per Model. (Currently not used, but will be used in future versions)
instance String Specifies that the getter returns a model instance. This needs to be specified if the View interacts with the returned model object. i.e. The View updates values on the model. Only available within a ViewModel
array String Specifies that the field holds an array. If the array hasn’t been initialized, it will automatically be initialized with an empty array.
pseudo String Specifies that the field does not retain it’s value and will be reset to undefined after each state transition. It’s value is usually derived from other fields.
static String Specifies that the field should not change and will persist it’s state during state transitions unless explicitly updated.

Usage

id: {
              kind: 'uid',
              get: function() { return this.state.id; }
            }

            selectedPerson: {
              kind: 'instance',
              get: function() { return this.state.selectedPerson; }
            }

            collection: {
              kind: 'array',
              get: function(){ return this.state.collection; },
            },

            selectedHobby: {
              kind: 'pseudo',
              get: function() {
                return this.state.hobbiesContext.current ?
                  this.state.hobbiesContext.current.name: void(0);
              }
            }
            // OR
            progress: {
              kind: 'pseudo',
              get: function() {
                return this.state.progress ? this.state.progress : 0;
              }
            }

            screenOrientation: {
              kind: 'static',
              get: function() {
                return this.state.screenOrientation;
              }
            }

#StateChangeHandler

Invoked when model fields change. i.e. setState was called within the model.

Signature

void <= StateChangeHandler(nextState[, callback])

Arguments

Param Type Details
nextState Object Next state of the Model instance
callback Function callback function passed in from the model

Usage

personStateChangeHandler: function(nextState){
              var persons = {};
              persons.collection = this.collection.map(function(person){
                if(person.id === nextState.id){
                  persons.selected = new Person(nextState, person, true);
                  return persons.selected;
                }
                return person;
              }.bind(this));
              this.setState(persons);
            },

Usage Context

optional :: ViewModel

                nextPath, prevPath){

                return { hobbies: { current: void(0) }, busy: false };
              }
            };

Instantiation

A user defined function is used to create model instances.

Please refer to Instantiating a Model instance for an explaination on why and how to structure your code to instantiate model instances.

Signature

ModelInstance <= #ModelConstructorClass([stateChangeHandler])([nextState, extendState, initialize])

Arguments

Param Type Details
stateChangeHandler (optional) Function Invoked whenever a model’s property changes. i.e. setState is called in the model. Only pass in stateChangeHandlers in ViewModels. You instantiate a Model within Models but you do not pass in a stateChangeHandler.
nextState (optional) Object Object containing the next state values. This must contain all the fields of the target model unless you are instantiating a new model. Then you only pass in the required fields.
extendState (optional) Object Object containing more next state values to be merged with nextState argument.
initialize (optional) Boolean true = Indicates that this is a new Model instance and that getInitialState should be called. (default = false unless no arguments are passed in the assumed to be initialising ie. initialize = true)

Usage

var Person: function(){
              return new PersonCtr(personStateChangeHandler).apply(this, arguments);
            },
