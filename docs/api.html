Astarisx.extend

Object function extend(Object sourceObj[, Object mergeObj ...])

Creates and returns a shallow copy of an object and merges the properties of any subsequent objects passed in. Arguments that are null or undefined are ignored.

Properties of preceding objects will be completely overwritten by a property with the same key in subsequent objects. The values are not merged.

Model Instantiation

When you define a Model, you're actually defining an object that returns a Model Constructor. The Model Constructor returns a Model Class, which is used to instantiate Model instances. See the Guide on usage.

Model Constructor

Model Class function modelCtor([Function modelChangeHandler])

Constructs a Model Class to be instantiated.

Model Class

Model Instance function modelClass([Object nextModelState][, Object extendModelState])

Instantiates a Model instance.

Enables automatic handling of pushState for <a> tags. There is no need to include this mixin in order to use the built-in pushState functionality. This mixin is only required if pushState is enable and <a> tags are used in the UI for navigation.

dataContextWillInitialize

void function dataContextWillInitialize()

Called before the data context is intialized (i.e. before getInitialState is called) and after it has been bootstrapped. This is where ajax calls are made to retrieve data and set up the data context. dataContextWillInitialize will only be called once per data context.

Scope: ControllerViewModel

initializeDataContext

void function initializeDataContext([
    String dataCtx ... |
    Object dataCtxInitObj ... |
    Mixed dataCtxs and dataCtxInitObjs ... |
    String Array dataCtxs |
    Object Array dataCtxInitObjs |
    Mixed Array dataCtxs and dataCtxInitObjs
])

Initializes the specified data context(s). This method will invoke dataContextWillInitialize for the specified data contexts. initializeDataContext can be invoked multiple times for the same data context, but dataContextWillInitialize will only be invoked the first time per data context.

Scope: ControllerViewModel

getWatchedState

Object function getWatchedState()

Watch fields and/or data contexts, specifying a stateChange handler, which is invoked if the watched state changes. See watchedStateChangeHandler.

Scope: ViewModel

getRoutes

Object function getRoutes()

Provides routing definition and specifies the routeChange handler invoked when the route is navigated to as a result of a popState Event. i.e Browser’s back or forward is clicked.

Scope: ViewModel

setState

void function setState([
    Object nextState,
    [Object nextAppState,]
    [Boolean remember = true,]
    [Function callback,]
    [Number delay]
])

Triggers a state transition. If no arguments are passed in, then the latest Application Data Context is returned to the View. If there is a callback and delay is not set, the state transition is batched until setState is called again. A call to setState with no arguments will suffice.

n.b. callback recieves two arguments: err, appCtx.

Scope: ControllerViewModel (nextAppState not supported), ViewModel, Model (remember and delay are not supported)

revert

void function revert([Function callback])

Transition Application Data Context to the previous state.

n.b. callback recieves two arguments: err, appCtx.

Scope: ControllerViewModel

advance

void function advance([Function callback])

Transition the Application Data Context back to the state, prior to reverting.

n.b. callback recieves two arguments: err, appCtx.

Scope: ControllerViewModel

mediaChangeHandler

void function mediaChangeHandler(
    String id,
    Object mql,
    Boolean initializing
)

Triggered for registered media queries, whenever a query notification is raised. To register a media query you must specify an id within the @madia query attribute. The format of the id is .media#[id], replacing [id] with whatever identifier you would like to reference that query. See Registering a media query in the Developers Guide for more information.

Scope: ControllerViewModel

Attributes

viewModel

viewModel: Object ViewModel

Specified as a property within a ControllerViewModel field, it associates a field with a ViewModel. The field is exposed to the View as a data context. The View interacts with the ViewModel by invoking methods and accessing properties via the data context.

Scope: ControllerViewModel

aliasFor

aliasFor: String sourceKey

Specified as a property within a Model field, it transforms a key from the original data source to the specified field name.

Scope: Model

kind

kind: String type

Specified as a property within a Model field, it assigns a field type.

Available Types

Type Details
uid (Model scope only) Identifies the unique identifier for the Model. Only available within a Model and should only specified once per Model. (Currently not used, but will be used in future versions)
instance (ViewModel scope only) Specifies that the getter returns a model instance. This needs to be specified if the View interacts with the returned model object. i.e. The View updates values on the model. Only available within a ViewModel
array Specifies that the field holds an array. If the array hasn’t been initialized, it will automatically be initialized with an empty array.
pseudo Specifies that the field does not retain it’s value and will be reset to undefined after each state transition. It’s value is usually derived from other fields.
static (ControllerViewModel scope only ??) Specifies that the field should not change and will persist it’s state during state transitions unless explicitly updated.
object
object:freeze
object:deepFreeze
array:freeze
array:deepFreeze

validate

validate: Function

Specified as a property within a Model field, it is a user defined function that returns a Boolean after validating a fields value.

$pushState

$pushState: Boolean

Toggles pushState. Only available if pushState routing is enabled. Default = true

Scope: ControllerViewModel

$forceReplace

$forceReplace: Boolean

Override default pushState behavior and perform a replace state. Only available if pushState routing is enabled. Default = false

If set to true it is not necessary to reset forceState back to false as it will revert to false once the operation has been performed.

Scope: ControllerViewModel

$enableUndo

$enableUndo: Boolean

Override default enableUndo behavior and allow for an adhoc enableUndo. This causes a previousState to be created with the ability to revert to it. If set to true it is not necessary to reset enableUndo back to false as it will revert to false once the operation has been performed. Default = false.

Scope: ControllerViewModel

$notify

$notify: String | String Array

Specify a $viewKey during a setState which is to be notified that there has been a state change. Only the Views with the specified $viewKeys will be notified.

n.b. The main UI container is given a default $viewKey of *.

Scope: ControllerViewModel

$owner

$owner: String

Specify a Models owner field.

Scope: Model

_[field]

When defining Model fields, prefix the field name with an underscore to prevent the field from being sent to the back end. This essentially hides it from the server creating a client-side or local field.

Scope: Model

View

initializeAppContext

void function initializeAppContext(
    Object appCtxInitObj
    [, Array dataCtxInitArg]
)

Initializes the Application Data Context.

Scope: UI

Param Details
appCtxInitObj Available Options:
  • controllerViewModel - Reference to ControllerViewModel
  • enableUndo - (Default = false); true: Enable undo\redo functionality.
  • enableRouting - (Default = false); true: Enable pushState routing and undo\redo functionality.
dataCtxInitArg

Specifies data contexts to be initialized. Can be an array of data context names AND/OR dataCtxInitObjs.

$viewKey

$viewKey: String

Passed to an Astarisx component. Used to identify a View if notify has been passed to setState. Please note that the component must be designated as a View by including the Astarisx.mixin.view mixin.

n.b. The UI container's viewKey is "*" (asterisk).

Scope: UI, View

.media#[key]

Unique key to identify a media query in a CSS file. The key is passed to mediaChangeHandler as the first parameter.
