Astarisx.extend

Creates and returns a shallow copy of an object and merges the properties of any subsequent objects passed in. Arguments that are null or undefined are ignored.

Properties of preceding objects will be completely overwritten by a property with the same key in subsequent objects. The values are not merged.

Usage

Astarisx.extend(obj[, objN])

Arguments

Param Type Details
obj Object An object that provides the initial properties to a shallow copy of the object.
objN (optional) Object Additional objects containing properties to merge into the target object.

Returns

Object - Shallow copy of merged object

initializeDataContext

Initializes the specified data context(s). This method will invoke dataContextWillInitialize for the specified data contexts.

n.b. initializeDataContext can be invoked multiple times for the same data context, but dataContextWillInitialize will only be invoked the first time per data context.

Usage

From ControllerViewModel 
this.initializeDataContext([dataCtxInitObjN]);
From a View Component 
this.state.appContext.initializeDataContext([dataCtxInitObjN]);

Arguments

Option 1

none

All data contexts are initialized.

Option 2

Param Type Details
dataCtxInitObjN String | Object | Mixed | Mixed Array

Specifies data contexts to be initialized. Can be "*" (asterisk) to initialize all data contexts OR a list of data context names AND/OR dataCtxInitObjs OR an array of data context names AND/OR dataCtxInitObjs.

An dataCtxInitObj has to following attributes: key = a data context name, value = arguments to be passed to dataContextWillInitialize for the specified data context.

There are 2 special key names that an dataCtxInitObj can have. They are an asterisk ("*") and an underscore followed by an asterisk ("_*"). The value for these keys specify the scope of the arguments. For example the asterisks indicates that all data contexts are to be intialized with the specified arguments. The key with the underscore followed by an asterisk specifies that all the data contexts list in the current initialization are to receive the specified arguments. It's like scoping the passed in args. Below is an example.

this.initializeDataContext({
  "*": ["globalArg1", "globalArg2"],
   "_*": ["contextArg1", "contextArg2"],
   "mockVM": ["mockVMArg1", "mockVMArg2"]
}, "hobbies");

The above example uses the asterisk to initialize all data contexts with the globalArg1 and globalArg2 arguments. Even the data contexts that are not listed. You will need to take care with this as once a data context has been initialized you are unable to initialize it again.

It uses the underscore and asterisk to initialize listed data contexts with the contextArg1 and contextArg2 arguments. mockVM and hobbies for this example.

mockVM will also receive its own arguments mockVMArg1 and mockVMArg2. mockVM and hobbies for this example.

The order in which the arguments are passed to datContextWillInitialize is Global, Context then Own. If any of the arguments are missing then they are simply not present. For example if there are no Context arguments then only Global and Own arguments will be present.

Returns

void

Scope

  • ControllerViewModel

setState

Triggers a state transition.

Usage

this.setState({ current: void(0) }, {
  busy: false,
  person: { online: true }
}, function(err, appContext){
  ...
  //setState must be called to update View.
  // It can be called without arguments eg. this.setState();
  this.setState({ current: aPerson });  
});

OR with delay parameter set

this.setState({fadeOut: true}, function(){
    this.setState({currentView: View});
}, 300); //delay callback by 300 milliseconds. nb. will not batch

Arguments

Param Type Details
nextState (optional) Object Object containing the next state values.
AppNextState (optional) Object Object containing the next state values for other data contexts. Only available in ViewModels
remember (default = true) Boolean false = Force the state controller not to retain previous state during this transition. Only available in the ControllerViewModel and ViewModels
callback (optional) Function Used to do sequential setState calls and is passed an error or the latest Application Data Context state. i.e. function(err, appCtx){…}
delay (optional) Number The number of milliseconds (thousandths of a second) that the callback function call should be delayed by. A callback must be specified. This will not batch updates.

If no arguments are passed in, then the latest Application Data Context is returned to the View.

If delay is not set, callbacks are batched until setState is called again. Therefore, setState must be called in a callback for the Data Context to transition to the next state. A call to setState with no arguments will suffice.

Returns

void

Scope

  • ViewModel

    return person;
  }.bind(this));

},

mediaChangeHandler

Triggered for registered media queries, whenever a query notification is raised. To register a media query you must specify an id within the @madia query attribute. The format of the id is .media#[id], replacing [id] with whatever identifier you would like to reference that query. See Registering a media query in the Developers Guide for more information.

Usage

mediaChangeHandler: function(id, mql, initializing){
    if(mql.matches){
      if(this.$canRevert){
        this.setState({mql:mql, media: id, $notify:'NavBarView'});
      } else {
        this.setState({mql:mql, media: id, $notify:'NavBarView'}, false);
      }
    }
}

Arguments

Param Type Details
id String Identifier associated with the calling media query.
mql Object MediaQueryList Object.
initializing Boolean true if initializing otherwise undefined.

Scope

  • ControllerViewModel

kind

Assigns a field type.

Attributes

Value Type Details
uid String Identifies the unique identifier for the Model. Only available within a Model and should only specified once per Model. (Currently not used, but will be used in future versions)
instance String Specifies that the getter returns a model instance. This needs to be specified if the View interacts with the returned model object. i.e. The View updates values on the model. Only available within a ViewModel
array String Specifies that the field holds an array. If the array hasn’t been initialized, it will automatically be initialized with an empty array.
pseudo String Specifies that the field does not retain it’s value and will be reset to undefined after each state transition. It’s value is usually derived from other fields.
static String Specifies that the field should not change and will persist it’s state during state transitions unless explicitly updated.

Usage

id: {
              kind: 'uid',
              get: function() { return this.state.id; }
            }

            selectedPerson: {
              kind: 'instance',
              get: function() { return this.state.selectedPerson; }
            }

            collection: {
              kind: 'array',
              get: function(){ return this.state.collection; },
            },

            selectedHobby: {
              kind: 'pseudo',
              get: function() {
                return this.state.hobbiesContext.current ?
                  this.state.hobbiesContext.current.name: void(0);
              }
            }
            // OR
            progress: {
              kind: 'pseudo',
              get: function() {
                return this.state.progress ? this.state.progress : 0;
              }
            }

            screenOrientation: {
              kind: 'static',
              get: function() {
                return this.state.screenOrientation;
              }
            }

Instantiation

A user defined function is used to create model instances.

Please refer to Instantiating a Model instance for an explaination on why and how to structure your code to instantiate model instances.

Signature

ModelInstance <= #ModelConstructorClass([modelChangeHandler])([nextState, extendState, initialize])

Arguments

Param Type Details
modelChangeHandler (optional) Function Invoked whenever a model’s property changes. i.e. setState is called in the model. Only pass in modelChangeHandlers in ViewModels. You instantiate a Model within Models but you do not pass in a modelChangeHandler.
nextState (optional) Object Object containing the next state values. This must contain all the fields of the target model unless you are instantiating a new model. Then you only pass in the required fields.
extendState (optional) Object Object containing more next state values to be merged with nextState argument.
initialize (optional) Boolean true = Indicates that this is a new Model instance and that getInitialState should be called. (default = false unless no arguments are passed in the assumed to be initialising ie. initialize = true)

Usage

var Person: function(){
              return new PersonCtr(personStateChangeHandler).apply(this, arguments);
            },

optional :: ViewModel, Model
