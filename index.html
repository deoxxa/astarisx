Astarisx

Highly Composable Application Architecture For React

Get Started

Currently v0.10.0

Productive

Leverage your React skills to start creating your next Astarisx application with little ramp up. That's because Astarisx feels like React, making it easy to comprehend and quick to learn. There's no need to change your coding style to suit the framework and it can easily be integrated into existing projects. Astatrisx comes with an easy to use, client-side push state router out of the box.

Highly Composable

Astarisx View's are stateless and completely decoupled from their Owner Component, making them highly composable. They can be moved to anywhere within your application and will work without changing a line of code. A View can even be completely removed without breaking your application or any interrelationships. State changes in one View are reflected in another.

Single Source of Truth

Application state is stored within ViewModels and Models. Astarisx handles state transitions, creating a Single Data Model, which is passed to the View and is accessible from anywhere within the application. This simplifies development. Invoke actions on data contexts from within the existing View or ViewModel. You are always garanteed that the View will have the latest copy of the Applications state when you need it.

Quick Overview

Astarisx uses the Model-View-ViewModel (MVVM) pattern, which was developed by Microsoft in 2005 and is a specialization of the Presentation Model. The View in MVVM is handled by React and is the User Interface (UI). Astarisx provides the framwork for the Models and ViewModels with which the UI will interact.

Models store the Application's Domain State and Data Domain Logic. A ViewModel is a "View's Model" and stores View State and Business Logic, referred to as a Data Context. The UI is completely decoupled from Business Logic and State.

Astarisx exposes Data Context's to the UI. The UI invokes actions on these Data Context's, causing the application to transition to a new State. A State Manager handles the state transition and passes the newly created state back to the UI, which reflects the state change. Every Astarisx View gets a complete copy of the new state. An Astarisx View is a self contained React component. They differ from regular React components, in that they are stateless and completely decoupled from their Owner, as they do not require any props to be passed in, making them highly composable.

The benefits of using Astarisx is that it's quick to learn and become productive. Astarisx Views are stateless and high composable, enabling the Separation of Concerns for easier code maintenance, collaboration and unit testing. And the Single Source of Truth application data context, accessible from anywhere within your application, makes Astarisx Applications easier to reason about.

/* Model */
var Todo = Astarisx.createModelClass({
  description: {
    get: function(){
      return this.$state.description;
    }
  }
});        

/* ViewModel */
var TodoClass = require("../models/todo");

// Model Constructor
var Todo = function(){
  return new TodoClass().apply(this, arguments);
};

var TodosViewModel = Astarisx.createViewModelClass({

  todosList: {
    kind: 'array',
    get: function(){
      return this.$state.todosList;
    }
  },

  addTodo: function(){
    var nextTaskList = this.$state.todosList.concat(new Todo({description: this.currentDescription}));
    this.setState({todosList: nextTaskList, currentDescription: ""});
  },

  currentDescription: {
    get: function(){
      return this.$state.currentDescription;
    },
    set: function(newVal){
      this.setState({currentDescription: newVal});
    }
  }
});
/* ControllerViewModel */
var ControllerViewModel = Astarisx.createCVMClass({
    //Specify a ViewModel's dataContext name
    todos: {
        viewModel: require('./todos'),
        get: function(){
            return this.$state.todos;
        }
    }
});
/* UI */
var TodoList = React.createClass({
  //Make component an Astarisx view
  mixins: [Astarisx.mixin.view],

  render: function(){
    //No props are passed in. All state is stored
    //in this.state.appContext
    var todos = this.state.appContext.todos;
    var list = todos.todosList.map(function(todo){
      return <li>{todo.description}</li>;
    });
    return <ul>{list}</ul>;
  }
});

var UI = React.createClass({
  //Make component an Astarisx UI
  mixins: [Astarisx.mixin.ui],
  
  componentWillMount: function(){
    //Associate ControllerViewModel with your app
    this.initializeAppContext({ 
      controllerViewModel: CVM
    });
  },
  
  addTodo: function(e){
    e.preventDefault();
    //Invoke an action on the todos dataContext
    this.state.appContext.todos.addTodo();
  },
  
  updateDescription: function(e){
    //Assign a value to currentDescription defined in the todos dataContext.
    //This would usually be a field in a Model. We are doing this
    //to keep the demo simple
    this.state.appContext.todos.currentDescription = e.target.value;
  },
  
  render: function(){
    return (<form>
      <fieldset>
        <label htmlFor="task">TODO</label>
        <input ref="inputFld" id="task" type="text" placeholder="Enter Task"
        onChange={this.updateDescription}
        value={this.state.appContext.todos.currentDescription} />
        <button onClick={this.addTodo}>Add</button>
      </fieldset>
      {/* Note: no props need to passed into the List component */}
      <List />
    </form>);
  }
});
